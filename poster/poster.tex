  \begin{alertblock}{Overview}
    We use a Minimum Description Length (MDL) framework, grounded in Occam's razor, 
    to discover a natural-language explanation $z$ for observed data $x$ that is both 
    concise and predictive. A frozen LLM scores candidate explanations, and this signal
    is used to train an explanation policy via reinforcement learning. 
    Our goal is an amortized system that can generate useful explanations for new datasets 
    without hand-crafted prompts.
  \end{alertblock}

  \begin{block}{Motivation}

    If prompts are viewed as natural-language models, then
    explanation discovery should not be treated as prompt
    tinkering alone. We want a system that can automatically
    identify explanations that capture the underlying pattern
    in the data rather than merely restating the observations.
  
    \begin{figure}
      \centering
      \begin{tikzpicture}[scale=6]
        \draw[step=0.25cm,color=gray] (-1,-1) grid (1,1);
        \draw (1,0) -- (0.2,0.2) -- (0,1) -- (-0.2,0.2) -- (-1,0)
          -- (-0.2,-0.2) -- (0,-1) -- (0.2,-0.2) -- cycle;
      \end{tikzpicture}
      \caption{A figure caption.}
    \end{figure}
  
    This requires balancing two competing goals: explanations
    should be simple enough to remain interpretable, but strong
    enough to predict the data well. That trade-off motivates
    our use of the Minimum Description Length principle.
  
  \end{block}

  \begin{block}{Minimum Description Length Framework}

    We model explanation discovery as selecting a natural-language
    hypothesis $z$ that gives the shortest description of the
    observed data $x$. The total score is a two-part code:
  
    \begin{equation*}
      \mathcal{L}_{\mathrm{MDL}}(z, x)
      =
      \underbrace{-\log p^*(z)}_{\text{cost of describing } z}
      +
      \underbrace{-\log p^*(x \mid z)}_{\text{cost of describing } x \text{ given } z}
    \end{equation*}

    \begin{itemize}
      \item Short explanations get a lower prior cost.
      \item Predictive explanations get a lower conditional data cost.
      \item The best explanation is the one that minimizes the
      total description length.
    \end{itemize}
  
  \end{block}

  \begin{block}{Method}

    Starting from a pretrained language model, we instantiate
    two copies: a frozen evaluator $p^*$ and a trainable policy
    $\pi_\theta(z \mid x)$ obtained by LoRA adaptation. Given
    observed data $x$, the policy samples a candidate explanation
    $z$, and the frozen evaluator scores it with an MDL-style reward
  
    \begin{equation*}
      R(z, x) = \lambda \log p^*(z) + \log p^*(x \mid z),
    \end{equation*}
  
    where the first term favors concise explanations under the
    model prior and the second favors explanations that make the
    observed data likely. In our implementation, $p^*(x \mid z)$
    is computed by conditioning on the candidate explanation and
    evaluating the likelihood of the observed target sequence.
  \end{block}
\end{column}

\begin{column}{\colwidth}
  \begin{block}{}
    The policy is optimized with REINFORCE:
    \begin{equation*}
      \mathcal{L}_{\mathrm{policy}}
      = -(R-b)\log \pi_\theta(z \mid x),
    \end{equation*}
    where $b$ is a moving-average baseline used to reduce gradient
    variance.
  
    The key idea is to learn an \emph{amortized} explanation policy
    while keeping the scoring model fixed. This preserves a stable
    MDL objective, reduces reward drift, and contrasts with
    per-dataset hypothesis search.
