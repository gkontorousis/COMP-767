\begin{block}{Motivation \& MDL Principle}

\begin{center}
\includegraphics[width=0.45\linewidth]{poster/resources/plot_good_fit.png}
\hfill
\includegraphics[width=0.45\linewidth]{poster/resources/plot_overfit.png}
\end{center}

\smallskip

\textbf{Which explanation would you trust?}

\smallskip

Most people prefer the simple quadratic fit (left) over the
highly complex curve (right), even though both match the data.

\smallskip

\textbf{Why?} We naturally apply \emph{Occam’s razor}:
prefer explanations that are both simple and predictive.

\smallskip

\textbf{But language models do not explicitly optimize for this.}
They favor explanations that are \emph{likely to be said}, not
necessarily those that best explain the data.

\smallskip

We formalize this intuition using the \textbf{Minimum Description Length (MDL)}
principle, selecting an explanation $z$ that minimizes:

\begin{equation*}
\mathcal{L}_{\mathrm{MDL}}(z, x)
=
\underbrace{-\log p^*(z)}_{\text{complexity}}
+
\underbrace{-\log p^*(x \mid z)}_{\text{data fit}}
\end{equation*}

\begin{itemize}
  \item Simple explanations $\rightarrow$ lower complexity cost
  \item Predictive explanations $\rightarrow$ lower data cost
  \item Best explanation $\rightarrow$ shortest total description
\end{itemize}

\end{block}

\begin{block}{Method}

  We use a pretrained LLM in two roles:
  a frozen evaluator $p^*$ and a trainable policy
  $\pi_\theta(z \mid x)$ (via LoRA).

  Given data $x$, the policy samples an explanation $z$,
  which is scored by an MDL-style reward:
  \begin{equation*}
    R(z, x) = \lambda \log p^*(z) + \log p^*(x \mid z).
  \end{equation*}

  The first term favors concise explanations,
  while the second favors explanations that predict the data.

  The policy is optimized with REINFORCE:
  \begin{equation*}
    \mathcal{L}
    = -(R - b)\log \pi_\theta(z \mid x),
  \end{equation*}
  where $b$ is a baseline.

  \smallskip

  \textbf{Key idea:}
  learn a policy that proposes explanations maximizing
  simplicity and predictive accuracy under a fixed evaluator.

\end{block}

\begin{block}{Guess the Loss!}

\small

Rank the explanations from \textbf{lowest MDL loss} (best) to highest.

\vspace{0.4em}

\textbf{Sequence A:} \\
$1, 1, 2, 5, 12, 27, 58, 121, \dots$

\begin{enumerate}
  \item $2^n - n$
  \item $a_n = a_{n-1} + a_{n-2}$
  \item \texttt{the pattern is [1, 1, 2, 5, 12, 27, 58, 121]}
\end{enumerate}

\vspace{0.5em}

\textbf{Sequence B:} \\
$6, 7, 13, 20, 33, 53, 86, 139, \dots$

\begin{enumerate}
  \item $a_n = a_{n-1} + a_{n-2}$
  \item $a_n = 2n + 5$
  \item \texttt{the pattern is [6, 7, 13, 20, 33, 53, 86, 139]}
\end{enumerate}

\vspace{0.6em}
\hrule
\vspace{0.4em}

\textbf{Answer (MDL ranking):}

\begin{itemize}
  \item \textbf{A:} $2^n - n$ $<$ Fibonacci $<$ memorization
  \item \textbf{B:} Fibonacci $<$ linear $<$ memorization
\end{itemize}

\textit{MDL prefers explanations that are both simple and predictive,
and penalizes memorization.}

\end{block}

\begin{block}{Training Loop}

  \begin{center}
    \includegraphics[width=0.98\linewidth]{poster/resources/flow.png}
  \end{center}

  \smallskip

  \begin{itemize}
    \item We generate a synthetic dataset of number sequences $x$ in Python.
    \item A LoRA-adapted policy $\pi_\theta(z \mid x)$ samples candidate explanations $z$.
    \item A frozen evaluator $p^*$ scores each explanation using the MDL objective:
    \[
      -\log p^*(z) - \log p^*(x \mid z)
    \]
    \item The same sampled explanation is also scored under the policy via
    $\log \pi_\theta(z \mid x)$.
    \item We combine both terms in a REINFORCE loss to update only the LoRA adapter weights.
  \end{itemize}

\end{block}

\begin{block}{Challenges}

\begin{itemize}

  \item \textbf{Likelihood estimation:}  
  Accurately computing $p(x \mid z)$ is difficult with small LMs.
  Without chain-of-thought, models struggle with precise reasoning
  (e.g., arithmetic), leading to noisy likelihood estimates.

  \item \textbf{Scaling to natural language:}  
  On real-world data (e.g., ScienceWorld-style traces), MDL scores
  become less meaningful. Token-level probabilities are diffuse,
  and differences outside the top predictions provide weak signal
  for explanation quality.

  \item \textbf{Exploration / discovery:}  
  Correct explanations often lie in low-probability regions of the
  model. Policy optimization improves local candidates but rarely
  discovers the true rule without guidance.

  \item \textbf{Ablation (oracle injection):}  
  Injecting the correct rule confirms that the reward and update
  can reinforce it once observed, indicating that the main bottleneck
  is exploration rather than reward design.

\end{itemize}

\end{block}

\textbf{Key observation:}
\begin{itemize}
  \item The base model prefers the max-likelihood recurrence rule.
  \item After fine-tuning, the policy assigns low conditional loss to the true rule.
  \item Fine-tuning moves model preference closer to the MDL objective.
\end{itemize}

\begin{block}{Conclusion}

MDL-based fine-tuning (via LoRA) shifts LLMs from producing likely text to generating better explanations by balancing plausibility $p(z)$ and predictive accuracy $p(x \mid z)$. This leads to concise, interpretable, and more correct rules, and extends beyond number sequences to general pattern discovery.

\end{block}
