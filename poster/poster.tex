  \begin{alertblock}{Overview}
    We use a Minimum Description Length (MDL) framework, grounded in Occam's razor, 
    to discover a natural-language explanation $z$ for observed data $x$ that is both 
    concise and predictive. A frozen LLM scores candidate explanations, and this signal
    is used to train an explanation policy via reinforcement learning. 
    Our goal is an amortized system that can generate useful explanations for new datasets 
    without hand-crafted prompts.
  \end{alertblock}

\textbf{Which model is a better explanation of the data?}

\smallskip

Most people prefer the simple quadratic fit (left) over the
highly complex curve (right), even though both can match the data.

\smallskip

\textbf{Why?} We naturally apply \emph{Occam’s razor}:
prefer explanations that are both simple and predictive.

\smallskip

This intuition is captured by the Minimum Description Length (MDL)
principle:

\begin{equation*}
\mathcal{L}(z, x) = -\log p(z) - \log p(x \mid z)
\end{equation*}

\begin{itemize}
  \item $-\log p(z)$: complexity of the explanation
  \item $-\log p(x \mid z)$: how well it fits the data
\end{itemize}

\textbf{Goal:} find explanations that balance simplicity and accuracy.

\end{block}

  \begin{block}{Minimum Description Length Framework}

    We model explanation discovery as selecting a natural-language
    hypothesis $z$ that gives the shortest description of the
    observed data $x$. The total score is a two-part code:
  
    \begin{equation*}
      \mathcal{L}_{\mathrm{MDL}}(z, x)
      =
      \underbrace{-\log p^*(z)}_{\text{cost of describing } z}
      +
      \underbrace{-\log p^*(x \mid z)}_{\text{cost of describing } x \text{ given } z}
    \end{equation*}

    \begin{itemize}
      \item Short explanations get a lower prior cost.
      \item Predictive explanations get a lower conditional data cost.
      \item The best explanation is the one that minimizes the
      total description length.
    \end{itemize}
  
  \end{block}

\begin{block}{Method}

  We use a pretrained LLM in two roles:
  a frozen evaluator $p^*$ and a trainable policy
  $\pi_\theta(z \mid x)$ (via LoRA).

  Given data $x$, the policy samples an explanation $z$,
  which is scored by an MDL-style reward:
  \begin{equation*}
    R(z, x) = \lambda \log p^*(z) + \log p^*(x \mid z).
  \end{equation*}

  The first term favors concise explanations,
  while the second favors explanations that predict the data.

  The policy is optimized with REINFORCE:
  \begin{equation*}
    \mathcal{L}
    = -(R - b)\log \pi_\theta(z \mid x),
  \end{equation*}
  where $b$ is a baseline.

  \smallskip

  \textbf{Key idea:}
  learn a policy that proposes explanations maximizing
  simplicity and predictive accuracy under a fixed evaluator.

\end{block}

$(1, 1, 2, 5, 12, 27, 58, 121, 248, 503)$.

\smallskip

\textbf{Key observation:}
\begin{itemize}
  \item Conditional likelihood favors what the model would say.
  \item MDL favors concise, predictive explanations.
\end{itemize}

\textbf{Training:}
We fine-tune a LoRA policy using REINFORCE with an MDL reward.
To improve exploration, we inject the correct rule with small
probability during sampling.

\smallskip

After fine-tuning, the model converges to the correct rule
$2^n - n$, achieving both low MDL loss and high confidence.

\end{block}

  \begin{block}{Conclusion}

  Fine-tuning a language model with an MDL-style reward
  (via LoRA) shifts it from predicting likely text to
  generating better explanations.

  Instead of optimizing only $p(z \mid x)$, the model is
  encouraged to balance:
  \begin{itemize}
    \item plausibility: $p(z)$
    \item correctness: $p(x \mid z)$
  \end{itemize}

  This leads to more concise, predictive, and interpretable
  rules, rather than fluent but incorrect explanations.

  The approach is domain-agnostic and can be applied beyond
  number sequences to general text-based pattern discovery.

\end{block}
